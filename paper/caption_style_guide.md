# Caption Style Guide

Use one caption house style across figures and tables.

## Structure

Each caption should answer, in order:

1. what it is
2. what it shows
3. why it matters for the paper’s argument

## Vocabulary rules

Reuse the canonical terms from [terminology_ledger.md](E:\Projects\StableSteering\paper\terminology_ledger.md):

- `traceable research platform`
- `prompt-first, preference-guided multi-round diffusion refinement`
- `steering state z`
- `qualitative case study`
- `replayable steering loop`

Avoid casual synonym drift in captions and labels.

## Evidence rules

- figures from docs assets should be described as schematics or workflow diagrams
- figures from runtime artifacts should be described as qualitative demonstration artifacts
- tables should name the evidence type first and avoid introducing new unsupported claims
