Claim-to-Evidence Matrix

Claim Status Evidence Notes / action
StableSteering is a prompt-first platform for iterative preference-guided diffusion image generation. verified README.md, app/frontend/templates/setup.html, app/main.py, app/engine/orchestrator.py Supported by UI flow and backend lifecycle.
The system maintains a low-dimensional steering state and updates it across rounds. verified app/core/schema.py, app/engine/orchestrator.py, docs/theoretical_background.md Supported as an implementation claim. Do not claim optimality of the dimension choice.
The repository includes a runnable application with replay, diagnostics, and trace artifacts. verified app/main.py, app/core/tracing.py, app/frontend/templates/replay.html, app/frontend/templates/diagnostics.html Strong systems claim.
The default runtime uses a real GPU-backed Diffusers backend. verified app/engine/generation.py, app/core/config.py, tests/test_generation_engine.py, tests/test_runtime_policy.py Good claim.
Per-session YAML configuration changes runtime behavior. verified app/core/config_yaml.py, app/core/schema.py, app/engine/generation.py, tests/test_config_yaml.py, docs/configuration_manual.md Strong claim.
The system supports multiple sampling, feedback, and update modules. verified app/samplers/random_local.py, app/samplers/axis_sweep.py, app/samplers/exploit_orthogonal.py, app/samplers/incumbent_mix.py, app/samplers/uncertainty.py, app/feedback/normalization.py, app/updaters/winner_copy.py, app/updaters/winner_average.py, app/updaters/linear_pref.py Supported as modularity, not superiority.
The first round includes a baseline prompt candidate and later rounds carry forward the previous winner. verified app/engine/orchestrator.py, tests/test_session_lifecycle.py Good differentiating implementation detail.
The repo contains a real multi-round example run with five rounds and twenty-five candidate presentations. verified output/examples/real_e2e_example_run/manifest.json, output/examples/real_e2e_example_run/real_e2e_example_run.html, scripts/create_real_e2e_example.py Safe as a qualitative case-study claim.
The repo includes an executed repeated-run pilot baseline matrix with 3 prompts, 3 policies, 27 runs, 36 rounds, and 144 candidate images. verified paper/results/baseline_matrix/manifest.json, paper/results/baseline_matrix/pilot_summary.md, scripts/run_paper_minimal_baseline_matrix.py Safe as a bounded repeated pilot-comparison claim. Do not present it as a broad benchmark.
The pilot matrix distinguishes workflow behavior across prompt-only, no-update, and StableSteering default policies. verified paper/results/baseline_matrix/tables/baseline_summary.csv, paper/results/baseline_matrix/tables/repeat_summary.csv, paper/results/baseline_matrix/pilot_table.md Safe if phrased as a bounded workflow comparison. Avoid outcome-superiority language.
The repeated pilot shows narrow workflow-level repeatability across three independent repeats per prompt-policy cell. verified paper/results/baseline_matrix/tables/repeat_summary.csv, paper/results/baseline_matrix/seed_robustness.md Safe only for repeated counts such as rounds/run and feedback events/run. Do not extend this to outcome quality.
The repo includes a paper-facing analysis layer that derives normalized workflow summaries from the repeated pilot bundle. verified paper/results/baseline_matrix/analysis/analysis_summary.md, paper/results/baseline_matrix/analysis/policy_summary.csv, scripts/build_paper_baseline_analysis.py Safe if phrased as descriptive workflow analysis only.
The repo includes a completed budget-normalized seed-policy slice with matched budgets across two steering-loop seed policies. verified paper/results/seed_policy_slice/README.md, paper/results/seed_policy_slice/analysis/analysis_summary.md, paper/protocols/budget_normalized_seed_policy_protocol.md Safe only as a bounded workflow comparison under matched budgets.
The repo includes a completed fixed-sampler updater ablation bundle with matched budgets across three updater choices. verified paper/results/updater_ablation/README.md, paper/results/updater_ablation/analysis/analysis_summary.md, paper/protocols/updater_ablation_protocol.md Safe only as a bounded modularity comparison.
The example run has passing image-sanity checks. verified output/examples/real_e2e_example_run/manifest.json, scripts/create_real_e2e_example.py Safe if phrased as automated sanity checks, not human quality evaluation.
The codebase includes broad backend and browser verification. verified tests/README.md, tests/test_session_lifecycle.py, tests/test_generation_engine.py, tests/test_feedback.py, tests/e2e/app.spec.js, tests/e2e/real_backend.spec.js Strong systems claim.
StableSteering is a configurable and reproducible research platform for studying interactive steering. partially supported docs/system_specification.md, docs/research_improvement_roadmap.md, app/core/tracing.py, app/storage/repository.py Supported as infrastructure. Not yet supported as a completed research evaluation platform with study data, benchmark corpus, or analysis notebooks.
The platform studies human-in-the-loop steering. should be softened docs/motivation.md, docs/research_improvement_roadmap.md Better phrasing: “is designed to support” or “is intended for” studying human-in-the-loop steering.
The method improves outcomes over rounds. partially supported output/examples/real_e2e_example_run/manifest.json, output/examples/real_e2e_example_run/real_e2e_example_run.html Only supported as one curated qualitative example.
The repo does not yet support strong claims of superiority over prompt-only baselines. verified docs/research_improvement_roadmap.md, paper/evidence_inventory.md Keep this limitation explicit.
The paper can make strong novelty claims today. unsupported paper/manuscript_draft.md, docs/research_improvement_roadmap.md The repo currently has no bibliography or citation base.

Most important softening actions

  • Replace “configurable and reproducible research platform” with “configurable and traceable platform intended for research on interactive steering”.
  • Replace “studying interactive steering” with “supporting future controlled studies of interactive steering”.
  • Replace any wording implying measured improvement over rounds with “illustrates a qualitative multi-round refinement trajectory”.
  • Replace any wording implying sampler or updater effectiveness with “implements interchangeable sampler and updater modules”.
  • Distinguish the curated qualitative case study from the executed pilot baseline matrix every time comparative evidence is discussed.
  • Treat the repeated pilot as workflow-level repeatability only unless and until paper-facing quality annotations or stronger evaluation metrics are added.
  • Treat derived rates in the analysis layer as descriptive workflow summaries, not outcome metrics.
  • Treat the seed-policy slice and updater ablation as controlled infrastructure evidence, not as algorithm-selection evidence.
